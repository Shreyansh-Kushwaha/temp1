# PTM AI Agent — Screens

Every screen the PTM Agent UI needs. Written before any code. Each entry lists: what it shows, what data it needs, and which API call powers it.

---

## Screen 1: Approval Queue — `/ptm`

**What it shows:**
Paginated table of PTM draft reports assigned to the logged-in teacher for the current reporting cycle. Header shows the reporting month.

**Data:**
- student_name, subject, reporting_month (formatted), status badge, generated_at (relative time), regeneration_count, report ID

**Status values:** `pending` · `approved` · `rejected` · `delivered` · `escalated`

**UI elements:**
- Orange reminder banner at top if any report has status=pending and age >24h: "You have [N] report(s) waiting — teachers aim to review within 24 hours."
- Table row actions: "Preview" (→ /ptm/[id]) + "Approve" quick-action button (only when status=pending)
- Status badge: pending=orange, approved=green, rejected=amber, delivered=teal, escalated=red
- Empty state: icon + "Nothing here yet — all caught up." + subtext
- Loading: skeleton rows with animate-pulse

**API call:** `GET /api/ptm/reports?teacher_id=&month=`

## Screen 2: Report Preview — `/ptm/[id]`

**What it shows:**
The full 1-page draft report exactly as the parent will receive it. Inferred fields are visually flagged.

**Report blocks (in order):**
1. **Header** — student name, subject, teacher name, reporting month, Sheldon Labs logo
2. **Sessions & Attendance** — total classes conducted, attendance %, no-shows count (3 stat cards)
3. **Learning Coverage** — bulleted list of topics covered this month
4. **Student Performance** — 1–3 sentence parent-friendly narrative
5. **Next Steps** — up to 4 bullet points for next month
6. **Teacher's Note** — optional italicised one-liner (empty placeholder if not added yet)

**Inferred field treatment:**
- Section background: `bg-ss-orange-50 border border-ss-orange-200 rounded-xl`
- Lucide `Info` icon + tooltip on hover: "Agent inferred this from class summaries — verify before approving"

**Actions:**
- "Approve" primary button (top-right) → opens teacher-note modal
- "Request Changes" secondary button → navigates to `/ptm/[id]/questionnaire`
- Back link → `/ptm`

**Teacher-note modal (on Approve):**
- Optional textarea: "Add a personal note for the parent (optional)"
- "Send Report" CTA → delivery confirmation toast → redirect to `/ptm`

**API calls:**
- `GET /api/ptm/reports/{id}`
- `POST /api/ptm/reports/{id}/approve` (with optional teacher_note body)

## Screen 3: Rejection Questionnaire — `/ptm/[id]/questionnaire`

**What it shows:**
A short, dynamic form (2–4 questions) targeting only the inferred fields the teacher disputes. Never shows all 6 question types at once.

**Question types (pulled from a fixed pool — shown only if relevant):**
- **Dropdown A — Engagement & Participation** (5 options)
- **Dropdown B — Concept Understanding** (5 options)
- **Dropdown C — Application & Practice** (5 options)
- **Topics covered correction** — free-text: "What topics did [student] actually cover this month?"
- **Next-month plan** — tag-style multi-select, max 4, validated against curriculum library
- **Free-form override note** — textarea: "Anything else the agent got wrong?"

**UI elements:**
- Progress indicator: "Question [X] of [N]"
- Each question on its own card (bg-white rounded-2xl shadow-ss p-6)
- Submit button → loading state: "Getting things ready…" (orange spinner)
- On success: redirect to `/ptm/[id]` to see the regenerated report

**API calls:**
- `GET /api/ptm/reports/{id}/questionnaire` → returns array of question objects to render
- `POST /api/ptm/reports/{id}/questionnaire` → submits answers, triggers regeneration

## Screen 4: Manager Escalation View — `/ptm/escalated`

**What it shows:**
All PTM reports that have hit the 2-cycle hard cap and require manual intervention.

**Data:**
- student_name, subject, teacher_name, reporting_month, escalated_at, regeneration_count (always 2)

**UI elements:**
- Section eyebrow: "ESCALATED" in red (`text-ss-error`)
- Each report as a card with red left border
- Actions per card: "Override & Deliver" (primary) | "Reassign" (secondary outline)
- Empty state: "No escalations — great work." with a success icon

**API calls:**
- `GET /api/ptm/escalated`
- `POST /api/ptm/escalated/{id}/override`

## Screen 5: Print / PDF Layout — `/ptm/[id]/print`

**What it shows:**
Clean, print-optimised version of the report — no navigation, no action buttons, no inferred-field highlights. This route is what Playwright renders to PDF.

**Layout:**
- Max-width 680px, centered, white background
- Sheldon Labs logo top-left, report month top-right
- Same 6 blocks as Screen 2 but plain — no orange highlights, no tooltips
- Footer: "Generated by Super Sheldon PTM Agent · Sheldon Labs"
- `@media print` hides everything outside the report container

**Data:** Same as Screen 2.
**API call:** `GET /api/ptm/reports/{id}` (reuses same endpoint, different render)
